**Severity:** Low (formatting only, no runtime impact)
**Files affected:**
- `apps/website/src/app/api/product/installation-flags/route.ts` -- CRLF line endings (must be LF)

**Reproduction:** From `C:/projects/grohe/NEO/grohe-neo-websites`, run:
```bash
pnpm biome check apps/website/src/app/api/product/installation-flags/route.ts \
  packages/core/api/src/product/getInstallationFlags.ts \
  packages/features/product-list/src/hooks/useRenderProductCard.tsx \
  packages/features/product/src/jss/PDPProductMain.server.tsx
```

**Fix:** Run `pnpm biome:fix` on these 4 files, or manually fix line endings to LF and
format the destructuring in `useRenderProductCard.tsx`.

### BUG-2: `isInstallationServiceIncluded` is hardcoded to `false` on PDP

**Severity:** Medium (potential missing UX element)
**File:** `packages/features/product/src/jss/PDPProductMain.server.tsx` line 503
**Current code:** `isInstallationServiceIncluded={false}`
**Expected:** For Zoovu "Buy It Installed" products, this should be `true` so the
"Installation included" label appears on the PDP. The property should likely be set to
`!!zoovuEnabledValue && (IsZoovuConfigurable as boolean)` (same gate as `isZoovuConfigurable`).

**Impact:** The "Installation included" label (controlled by `instalationServiceLabel` in
`PDPPurchase.client.tsx` line 456-460) will never appear for any product, even if it is a
Zoovu "Buy It Installed" product.

**Note:** This may be intentional if the PO decided to defer this label to a future ticket.
The dev agent should clarify with the PO whether this label should be shown for Zoovu products.

## 5. Recommendations for Dev Agent

1. **Fix Biome formatting (BUG-1):** Run `pnpm biome:fix` on the 4 affected files to fix
   CRLF line endings and destructuring format.

2. **Clarify `isInstallationServiceIncluded` (BUG-2):** Confirm with PO whether the
   "Installation included" label should appear on PDP for Zoovu products. If yes, change
   `isInstallationServiceIncluded={false}` to
   `isInstallationServiceIncluded={!!zoovuEnabledValue && (IsZoovuConfigurable as boolean)}`
   on line 503 of `PDPProductMain.server.tsx`.

3. **`ProductsApiBaseUrl` config:** Ensure `appsettings.Testing.json` / `appsettings.Staging.json` /
   `appsettings.Production.json` for ShoppingCartApi include the new
   `HttpConfiguration:ProductsApiBaseUrl` key pointing to the ProductsApi Cloud Run service URL.
   Without this, `IProductsApi` will not be registered and `InstallationFlagService` will
   default to `HasInstallationProduct = false` silently.
