# QA Report: NEO-4928 -- Navigation Settings Error URL Fields

**Date:** 2026-03-12
**Ticket:** NEO-4928
**Feature:** dynamic-navigation
**Verdict:** PARTIAL -- implementation is correct but depends on missing Sitecore template fields

## Summary

Two new fields (`Error404Url`, `Error500Url`) were added to the XmCloudApi navigation-settings
endpoint. The .NET implementation is correct, consistent with all existing field mappings, and
the project builds cleanly with zero warnings. However, the Sitecore XM Cloud template
**Navigation Settings** does not yet contain these two fields in the serialized items, and the
frontend (`grohe-neo-websites`) does not yet consume them. These are expected downstream tasks
but should be tracked.

Current fields (16 total):

`PrivacyPolicyPage`, `ProductDetailsPage`, `ProductListingPage`, `ProductsCatalogUrl`,
`RegistrationUrl`, `SearchResultPageUrl`, `SearchUrl`, `ShoppingCartPage`,

(Plus two in an alternate path: `LoggedInProjectListExportUrl`, `TrainingCourseRegistrationFormPage`)

**`Error404Url` and `Error500Url` are NOT present** in the serialized Sitecore items.

This means:
1. Until these fields are added to the Sitecore template and populated with page references,
   the GraphQL response will never contain fields named `Error404Url` or `Error500Url`.
2. The API will return `null` for both fields, which is safe (nullable properties).
3. The Sitecore template change is likely a separate task or already done directly in XM Cloud
   without being serialized back to the repo.

**Recommendation:** Confirm with the team whether these fields have been added directly in
Sitecore XM Cloud (Content Editor or XM Cloud Deploy) and just need serialization pull, or
whether a separate Sitecore task is needed to create them.
